# 사용 가능한 환경

다음은 Isaac Lab에서 사용 가능한 모든 RL 및 IL 작업 구현 목록입니다.
목록을 최신 상태로 유지하려고 노력하지만, 다음 명령을 실행하여 언제든지 최신 환경 목록을 얻을 수 있습니다.

### Linux

#### NOTE
키워드 검색을 위해 `--keyword <search_term>` 옵션을 사용할 수 있습니다(선택 사항).

```bash
./isaaclab.sh -p scripts/environments/list_envs.py --keyword <search_term>
```

### Windows

#### NOTE
키워드 검색을 위해 `--keyword <search_term>` 옵션을 사용할 수 있습니다(선택 사항).

```batch
isaaclab.bat -p scripts\environments\list_envs.py --keyword <search_term>
```

환경 목록을 계속 확장하고 있습니다. Isaac Lab에 추가하고 싶은 환경이 있다면 언제든지 풀 리퀘스트를 열어주세요!

### 접촉이 풍부한 조작

페그 삽입, 기어 맞물림, 너트-볼트 체결과 같은 접촉이 풍부한 조작 작업을 기반으로 한 환경입니다.

이러한 작업은 동일한 작업 구성과 제어 옵션을 공유합니다. 작업 이름을 지정하여 전환할 수 있습니다.

### AutoMate

소켓에 플러그를 삽입하는 100가지 다양한 조립 작업으로 구성된 환경입니다. 각 작업은 부품의 기하학적 형태와 속성에 따라 다릅니다. 모든 작업은 동일한 구성을 공유합니다.

해당하는 asset ID를 지정하여 작업을 전환할 수 있습니다. 가능한 asset ID는 다음과 같습니다:

‘00004’, ‘00007’, ‘00014’, ‘00015’, ‘00016’, ‘00021’, ‘00028’, ‘00030’, ‘00032’, ‘00042’, ‘00062’, ‘00074’, ‘00077’, ‘00078’, ‘00081’, ‘00083’, ‘00103’, ‘00110’, ‘00117’, ‘00133’, ‘00138’, ‘00141’, ‘00143’, ‘00163’, ‘00175’, ‘00186’, ‘00187’, ‘00190’, ‘00192’, ‘00210’, ‘00211’, ‘00213’, ‘00255’, ‘00256’, ‘00271’, ‘00293’, ‘00296’, ‘00301’, ‘00308’, ‘00318’, ‘00319’, ‘00320’, ‘00329’, ‘00340’, ‘00345’, ‘00346’, ‘00360’, ‘00388’, ‘00410’, ‘00417’, ‘00422’, ‘00426’, ‘00437’, ‘00444’, ‘00446’, ‘00470’, ‘00471’, ‘00480’, ‘00486’, ‘00499’, ‘00506’, ‘00514’, ‘00537’, ‘00553’, ‘00559’, ‘00581’, ‘00597’, ‘00614’, ‘00615’, ‘00638’, ‘00648’, ‘00649’, ‘00652’, ‘00659’, ‘00681’, ‘00686’, ‘00700’, ‘00703’, ‘00726’, ‘00731’, ‘00741’, ‘00755’, ‘00768’, ‘00783’, ‘00831’, ‘00855’, ‘00860’, ‘00863’, ‘01026’, ‘01029’, ‘01036’, ‘01041’, ‘01053’, ‘01079’, ‘01092’, ‘01102’, ‘01125’, ‘01129’, ‘01132’, ‘01136’.

분해 및 조립을 위한 환경을 모두 제공합니다.

#### 주의
Nvidia 드라이버 570을 사용하는 Linux x86_64 아키텍처에서 AutoMate 환경을 실행할 때는 CUDA 12.8을 권장합니다. 아래 단계를 따라 CUDA 12.8을 설치하면 AutoMate 환경에서 CUDA를 사용해 보상을 계산할 수 있습니다. 다른 운영 체제나 아키텍처를 사용하는 경우, [CUDA 설치 페이지](https://developer.nvidia.com/cuda-12-8-0-download-archive)를 참조하여 추가 지침을 확인하세요.

```bash
wget https://developer.download.nvidia.com/compute/cuda/12.8.0/local_installers/cuda_12.8.0_570.86.10_linux.run
sudo sh cuda_12.8.0_570.86.10_linux.run --toolkit
```

conda를 사용할 경우, 다음과 같이 cuda 툴킷을 설치할 수 있습니다:

```bash
conda install cudatoolkit
```

Nvidia 드라이버 580과 CUDA 13을 사용할 때는 현재 AutoMate 환경에서 CUDA를 사용해 보상을 계산할 수 없습니다. 코드가 자동으로 CPU로 폴백되어 성능이 약간 낮아집니다.

* [Isaac-AutoMate-Disassembly-Direct-v0](https://github.com/isaac-sim/IsaacLab/blob/main/source/isaaclab_tasks/isaaclab_tasks/direct/automate/disassembly_env_cfg.py): 플러그가 소켓에 삽입된 상태에서 시작됩니다. 저레벨 컨트롤러가 플러그를 들어 올려 랜덤한 위치로 이동시킵니다. 이 과정은 완전히 스크립트로 제어되며 학습된 정책을 사용하지 않으므로 정책 훈련이나 평가가 필요하지 않습니다. 생성된 궤적은 역과정, 즉 조립을 배우기 위한 시연으로 사용됩니다. 특정 작업에 대한 분해를 실행하려면 다음 명령을 사용합니다: `python source/isaaclab_tasks/isaaclab_tasks/direct/automate/run_disassembly_w_id.py --assembly_id=ASSEMBLY_ID --disassembly_dir=DISASSEMBLY_DIR`. 모든 생성된 궤적은 로컬 디렉터리 `DISASSEMBLY_DIR`에 저장됩니다.

  * 조립 정책을 훈련하려면 다음 명령을 실행합니다: `python source/isaaclab_tasks/isaaclab_tasks/direct/automate/run_w_id.py --assembly_id=ASSEMBLY_ID --train`. 선택적 플래그를 사용하여 훈련 과정을 커스터마이즈할 수 있습니다: `--headless`는 GUI 창 없이 실행, `--max_iterations=MAX_ITERATIONS`는 훈련 반복 횟수 설정, `--num_envs=NUM_ENVS`는 훈련 중 병렬 환경 수 설정, `--seed=SEED`는 랜덤 시드 지정. 정책 체크포인트는 훈련 중에 자동으로 `logs/rl_games/Assembly/test` 디렉터리에 저장됩니다.
  * 조립 정책을 평가하려면 다음 명령을 실행합니다: `python source/isaaclab_tasks/isaaclab_tasks/direct/automate/run_w_id.py --assembly_id=ASSEMBLY_ID --checkpoint=CHECKPOINT --log_eval`. 평가 결과는 `evaluation_{ASSEMBLY_ID}.h5`에 저장됩니다.

### FORGE

FORGE 환경은 다음 기능을 추가하여 Factory 환경을 확장합니다:

* 힘 감지: 엔드 이펙터가 경험하는 힘에 대한 옵저베이션 추가.
* 과도한 힘 패널티: 과도한 접촉 힘에 대해 에이전트를 패널티화하는 옵션 추가.
* 다이내믹스 랜덤화: 컨트롤러 게인, 자산 속성(마찰, 질량) 및 데드존 랜덤화.
* 성공 예측: 작업 성공을 예측하는 추가 액션 추가.

이러한 작업들은 동일한 작업 구성 및 제어 옵션을 공유합니다. 작업 이름을 지정하여 전환할 수 있습니다.
